## 2. Clock Configuration (RCC)

The STM32F3 series has a simpler PLL structure than the F4/G4 series. It **does not** have PLLP, PLLQ, or PLLR. It only uses `PLLMUL` and `PLLDIV`.

1. Go to **Pinout & Configuration → System Core → RCC**.
    - High Speed Clock (HSE): **Crystal/Ceramic Resonator**.
2. Go to the **Clock Configuration** tab.
    - Input frequency: `8 MHz` (HSE).
    - PLL Source Mux: `HSE`.
    - **PLLMUL**: `x16`.
    - **PLLDIV**: `/2`.
    - System Clock Mux: `PLLCLK`.
3. Verify the resulting clock tree:
    - **HCLK = 64 MHz**
    - **APB1 Peripheral clocks = 32 MHz** (Prescaler /2)
    - **APB1 Timer clocks = 64 MHz** _(Hardware auto-multiplies by 2 when APB1 divider is not 1)_
    - **APB2 Peripheral clocks = 64 MHz**
    - **APB2 Timer clocks = 64 MHz**

## 3. I2C1 — AS5600 Magnetic Encoder

3. **Parameter Settings**:

    - PB7 → I2C1_SDA

## 4. TIM2 — Stepper STEP Pulse Generator

Used to hardware-generate the step pulses for the A4988. TIM2 is on APB1, so its timer clock is 64MHz.

1. Go to **Timers → TIM2**.
2. Mode: **PWM Generation CH1**.
3. **Pinout**: Verify PA0 → TIM2_CH1.
4. **Parameter Settings**:
    - Prescaler (PSC): `63` _(64MHz / 64 = 1MHz tick rate)_
    - Counter Mode: `Up`
    - Counter Period (ARR): `999` _(Placeholder, updated at runtime)_
    - auto-reload preload: `Enable`
5. **PWM Generation Channel 1**:
    - Mode: `PWM mode 1`
    - Pulse (CCR1): `499` _(Placeholder = ARR/2 for 50% duty)_
    - Output compare preload: `Enable`
6. **NVIC Settings**: Enable **TIM2 global interrupt** (Required for stepper ramp/step counting).

---

## 5. TIM3 — DC Spool Motor PWM (TB9051FTG)

The TB9051FTG has a dedicated PWM pin. IN1 and IN2 are standard logic GPIOs. Therefore, we **only use Channel 1** for PWM.

1. Go to **Timers → TIM3**.
2. Mode: **PWM Generation CH1** _(Do NOT select CH1 CH2)_.
3. **Pinout**: Verify PA6 → TIM3_CH1.
4. **Parameter Settings**:
    - Prescaler (PSC): `0` _(Timer clock = 64MHz)_
    - Counter Mode: `Up`
    - Counter Period (ARR): `3199` _(64MHz / 3200 = **20 kHz PWM**)_
    - auto-reload preload: `Enable`
5. **PWM Generation Channel 1**:
    - Mode: `PWM mode 1`
    - Pulse (CCR1): `0` _(Set at runtime)_
    - Output compare preload: `Enable`

## 6. USART2 — Debug Console

**Pin Conflict Resolution:** The original spec used PA2 for the Stepper Enable pin. Because USART2_TX requires PA2, the Stepper Enable pin was moved to PB1.

1. Go to **Connectivity → USART2**.
2. Mode: **Asynchronous**.
3. **Parameter Settings**:
    - Baud Rate: `115200`
    - Word Length: `8 Bits`
    - Parity: `None`
    - Stop Bits: `1`
4. **Pinout**: Verify PA2 → USART2_TX, PA3 → USART2_RX.

Configure the remaining logic pins by clicking on the pin in the chip diagram and selecting `GPIO_Output`.

|---|---|---|---|
|**PA1**|`DIR`|Low|Stepper direction (Active High = Reverse)|

## 8. NVIC Configuration

1. Go to **System Core → NVIC**.
2. Set the preemption priorities to ensure time-critical interrupts fire first:
    - **TIM2 global interrupt**: Preemption `1` (Stepper timing)
    - **I2C1 global interrupt**: Preemption `2` (Encoder reads - currently unused but set for future)
    - **USART2 global interrupt**: Preemption `3` (Debug - lowest priority)

## 10. Post-Generation Code Integration

The custom driver files (`as5600.c/h`, `stepper.c/h`, `dc_motor.c/h`) have been written specifically for this pinout. However, one manual code change is required in the generated code:

**In `stm32f3xx_it.c`:** You must call the stepper ISR inside the TIM2 Update interrupt handler. Add the include and the function call inside the `USER CODE` blocks:

c

/* USER CODE BEGIN Includes */

#include "stepper.h"

/* USER CODE END Includes */

  

/* USER CODE BEGIN EV */

extern Stepper_Handle_t hstepper;

/* USER CODE END EV */

  

void TIM2_IRQHandler(void)

{

/* USER CODE BEGIN TIM2_IRQn 0 */

/* USER CODE END TIM2_IRQn 0 */

HAL_TIM_IRQHandler(&htim2);

/* USER CODE BEGIN TIM2_IRQn 1 */

Stepper_ISR(&hstepper); // Crucial: Updates step counter

/* USER CODE END TIM2_IRQn 1 */

}
